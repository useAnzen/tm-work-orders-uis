Mobile Work Order Creation

1. Business Case ▼

Field workers need to create T&M work orders on-site, from their phones, at the end of a shift or when extra work is performed. This is the single most critical mobile feature for T&M workflow adoption. Without it, field data capture falls back to paper forms and WhatsApp photos — the exact problem Appello is solving.

A gap analysis against Clearstory revealed 24 feature gaps in the original 6-step wizard design. This revised spec restructures the flow into a 5-step wizard + Summary page + separate Prepare Signatures flow, achieving full feature parity with Clearstory while preserving Appello’s mobile-first design language.

Customer Evidence

2. Scope ▼

In Scope

  • IN 5-step wizard + Summary page + Prepare Signatures sub-flow: Info, Labor, Material, Equipment, Summary → Prepare Signatures
  • IN Project/Job title picker with inline project creation and full-screen search
  • IN Cost code field (optional text input)
  • IN T&M Tag Title field (maps to Job.name)
  • IN Rich text description editor (full-screen with Bold, Italic, Underline, Lists, Indent, Clear Formatting, DONE button)
  • IN Multi-date “Dates of Work Performed” (green pill chips with + to add, × to remove)
  • IN Photos & Documents section on Step 1 (0/50 limit, separate Photo/Document subsections, Camera/Photo Library/Files sources, JPEG/PNG/PDF support)
  • IN EXIT/SAVE header pattern (replaces back arrow + step counter)
  • IN Labor: “Qty of Workers” multiplier column
  • IN Labor: 5 hour columns — ST, OT, DT, Premium Time, Premium OT (horizontal scroll)
  • IN Labor: per-row Date of Work picker for each labor entry
  • IN Labor: “Add Worker” from imported rates sub-screen with empty state
  • IN Material & Equipment: 3-way add bottom sheet (“Select from Catalog”, “Add Blank Row”, “Bulk Edit” Phase 2)
  • IN Material & Equipment: import rate sub-screens with empty states
  • IN Full-width Previous Step / Next Step navigation buttons on all steps
  • IN Summary page (Step 5): full read-only recap of all entered data
  • IN “Other” line-item category (4th type: Name, Qty, UoM, Qty of Unit)
  • IN “Preview Tag” button with in-app PDF preview
  • IN “Prepare Signatures” as separate flow accessed from Summary
  • IN Signer management: add/select signers, Required/Optional types, search existing or create new (First Name, Last Name, Email, Company, Signer Type)
  • IN Signing order toggle
  • IN Email message to signers (collapsible textarea)
  • IN “Send signed copy to” distribution list
  • IN “Collect Signatures” button (distinct from Submit)
  • IN Draft management — save and resume multiple concurrent drafts
  • IN Auto-pricing based on selected trade levels and hour types

Out of Scope

  • OUT Full offline mode with background sync (Phase 2)
  • OUT Bulk Edit on material/equipment (Phase 2 — bottom sheet shows option but disabled)
  • OUT Barcode/QR scanning for materials
  • OUT Voice-to-text for description fields
  • OUT Desktop WO creation (covered in UIS-20c)

7. Acceptance Criteria ▼

Flow & Navigation

  • User sees a 5-step wizard (Info, Labor, Material, Equipment, Summary) with EXIT/SAVE header pattern on every step.
  • Progress bar updates correctly across all 5 steps (20%, 40%, 60%, 80%, 100%).
  • Full-width “Previous Step” and “Next Step” navigation buttons appear on all applicable steps.
  • EXIT button prompts save confirmation; SAVE button persists current draft without leaving.

Step 1: Info

  • Project/Job Picker is the first field, required. Opens full-screen picker with search and CREATE NEW PROJECT option.
  • Inline project creation works: user can enter project name and company, and the new project appears selected.
  • Cost Code field is optional text input.
  • T&M Tag Title field is present and maps to Job.name.
  • Description field opens a full-screen rich text editor with Bold, Italic, Underline, Ordered List, Unordered List, Indent, Outdent, Clear Formatting, and DONE button.
  • Multi-date “Dates of Work Performed” displays as green pill chips. User can add dates with + button and remove with ×.
  • Photos & Documents section shows on Step 1 with 0/50 counter, separate Photo and Document subsections.
  • Photo/document upload supports Camera, Photo Library, and Files sources. Accepts JPEG, PNG, and PDF.

Step 2: Labor

  • “Qty of Workers” column is present and multiplies the labor total.
  • 5 hour columns are displayed: ST, OT, DT, Premium Time, Premium OT. Table scrolls horizontally on narrow screens.
  • Each labor row has its own Date of Work picker.
  • “Add Worker” button opens imported rates sub-screen showing available trade levels with ST/OT/DT rates.
  • Empty state is shown on the Add Worker sub-screen when no trade-level rates are configured.
  • Labor item auto-calculates total based on trade level rates, hours, and worker count.

Steps 3 & 4: Material & Equipment

  • Tapping “+ Add Material” or “+ Add Equipment” opens a 3-way bottom sheet with: Select from Catalog, Add Blank Row, and Bulk Edit (Phase 2, disabled).
  • “Select from Catalog” opens an import rate sub-screen with search and item list.
  • Empty state is shown on catalog sub-screens when no rates are imported.
  • “Add Blank Row” creates an empty row for manual entry.

Step 5: Summary

  • Summary page displays a full read-only recap of all entered data: Info, Labor, Material, Equipment, and Other items.
  • “Other” line-item category is supported as a 4th item type with Name, Qty, Unit of Measure, and Qty of Unit fields.
  • Grand total is calculated and displayed across all line-item categories.
  • “Preview Tag” button generates and opens an in-app PDF preview of the T&M tag.
  • “Prepare Signatures” button navigates to the signature preparation flow.

Prepare Signatures

  • Prepare Signatures is a separate flow accessed from Summary, NOT a wizard step.
  • “Sign yourself” checkbox is available for the creator to include their own signature.
  • User can add signers via “Select a Signer” tab (search existing) or “Add a Signer” tab (create new with First Name, Last Name, Email, Company, Signer Type).
  • Each signer is tagged as REQUIRED or OPTIONAL.
  • Signing Order toggle enables/disables ordered signature collection. When enabled, signers can be reordered.
  • Collapsible email message textarea allows custom message to signers.
  • “Send signed copy to” distribution list allows adding email recipients who receive the final signed PDF.
  • “Collect Signatures” button triggers signature collection (distinct from Submit).
  • “Back to Summary” button navigates back to the Summary step.

General

  • User can save a draft at any step and resume it later; multiple concurrent drafts are supported.
  • Form handles validation errors inline (required fields highlighted, error messages displayed).
  • On submission, a notification is sent to the designated supervisor.
